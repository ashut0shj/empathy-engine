# Empathy Engine

A command-line and web application that reads text, detects its emotional tone, and generates speech audio that sounds different depending on whether the text is positive, negative, or neutral. Happy text is spoken faster and at a higher pitch. Sad text is spoken slower and quieter. Neutral text uses default voice settings.

---

## Technology Used

- **VADER** (vaderSentiment) - emotion detection from text. Primary choice.
- **TextBlob** - fallback emotion detection if VADER is unavailable.
- **pyttsx3** - offline text-to-speech engine. Controls speech rate and volume natively.
- **gTTS** (Google Text-to-Speech) - fallback TTS if pyttsx3 fails. Requires internet.
- **pydub** - post-processes the generated audio to apply pitch shifting, since pyttsx3 does not support pitch natively.
- **Flask** - lightweight web server for the optional browser interface.

---

## Setup

### Requirements

- Python 3.8 or higher
- ffmpeg installed on your system (required for pitch shifting via pydub)

Install ffmpeg:

- Ubuntu/Debian: `sudo apt install ffmpeg espeak libespeak1`
- Windows: Download from https://ffmpeg.org and add to your system PATH. pyttsx3 uses the built-in SAPI5 voice so no extra TTS install is needed on Windows.

### Install Python dependencies

```bash
cd empathy-engine

python -m venv venv
source venv/bin/activate        # Linux
venv\Scripts\activate           # Windows

pip install -r requirements.txt

python -m textblob.download_corpora
```

## Running the Application

### Single text input

```bash
python app/main.py --text "I am so happy today, everything is going great!"
python app/main.py --text "I feel completely exhausted and hopeless."
python app/main.py --text "The report is due on Friday."
```

Audio is saved to the `output/` folder.

### Interactive mode

```bash
python app/main.py --interactive
```

Type any text and press Enter. The engine detects the emotion, prints the voice parameters it chose, generates the audio, and saves the file. Type `quit` to exit.

### Run the demo

```bash
python demo_script.py
```

Runs five pre-written examples covering happy, sad, neutral, angry, and enthusiastic text. Prints a summary table showing detected emotion, confidence score, and chosen voice parameters for each.

To see the detection results without generating audio:

```bash
python demo_script.py --no-audio
```

### Web interface

```bash
python app/web_app.py
```

Open http://localhost:5000 in a browser. Enter text, click the button, and the page shows the detected emotion, confidence score, voice parameters used, and plays the generated audio.

### API

With the web server running:

```bash
curl -X POST http://localhost:5000/api/synthesize \
  -H "Content-Type: application/json" \
  -d '{"text": "This is wonderful news!"}'
```

Response includes emotion label, confidence score, raw sentiment scores, voice parameters, and a URL to download the audio file.

## Custom Voice Parameters

Edit `config/voice_config.json` to change how the voice sounds for each emotion. The three fields that matter most are:

- `rate` - words per minute. Higher is faster.
- `volume` - between 0.0 and 1.0.
- `pitch` - semitone shift applied to the audio. Positive is higher, negative is lower.

Pass the config file explicitly:

```bash
python app/main.py --text "Hello world" --config config/voice_config.json
```

## Design Notes

### Emotion detection

VADER was chosen over a HuggingFace transformer model for two reasons. First, it requires no model download and works offline. Second, it starts up instantly, which matters for a CLI tool. VADER gives a compound score between -1.0 and 1.0. Scores above 0.05 are classified as positive, below -0.05 as negative, and anything in between as neutral. TextBlob uses the same threshold logic on its polarity score and serves as a drop-in fallback.

### Emotion to voice mapping

The mapping is straightforward: positive text sounds energetic, negative text sounds subdued, neutral text sounds like a default voice.

- Positive: rate 210 wpm, volume 1.0, pitch +2 semitones
- Negative: rate 135 wpm, volume 0.75, pitch -3 semitones
- Neutral: rate 175 wpm, volume 0.9, pitch 0 semitones

The voice parameters are not applied as a hard switch. They are blended with the neutral defaults based on the confidence score from the detector. If VADER is 90% confident the text is positive, the final rate is calculated as: neutral_rate + 0.9 * (positive_rate - neutral_rate). This prevents over-modulation when the detector is uncertain.

### Pitch shifting

pyttsx3 controls rate and volume but has no pitch API. Pitch is applied after the audio is generated by changing the sample rate of the WAV file using pydub. Raising the sample rate before playback makes the audio sound higher in pitch without stretching the duration. This avoids adding a dependency on a neural audio model and runs fast on any hardware.

### TTS fallback chain

pyttsx3 is tried first because it works offline. If it fails (missing espeak on Linux, or driver issues), gTTS is used. gTTS only supports normal and slow speed modes, so negative emotion maps to slow and everything else maps to normal. Pitch shifting still applies to gTTS output.
